### ❌ Task 2.3: Model Evaluation & Testing — 500-Question Benchmark CANCELLED
**Status**: CANCELLED — benchmark design was invalid

**Why it was cancelled**: The 500-question benchmark (`data/eval/iks_benchmark_gold.json`) was generated by Gemini API producing questions from 20 held-out IKS documents. The intent was to then have the *fine-tuned Bharat model* answer those questions. But this design has a fatal circular validation flaw:

- Gemini wrote the questions
- Gemini's own knowledge of those documents shaped how the questions were framed
- Running Gemini again as the judge (LLM-as-a-judge) would mean Gemini scoring answers to questions it wrote, against answers it would have given

**The 100 hand-written and 200 adversarial questions described in `evaluation_framework.md` were never created.** Only the 200 Gemini-generated held-out questions exist, and they cannot be used as a valid benchmark for the reasons above.

**What a valid replacement benchmark looks like (for V2 evaluation)**:
- [ ] Write ~50 questions manually from primary IKS sources (not AI-generated)
- [ ] Use the 150-prompt regression suite (`data/eval/v2_regression_tests.jsonl`) for objective constraint testing (this is model-agnostic — no circular dependency)
- [ ] For subjective quality: have a human reviewer score a random sample of 30 Bharat V2 responses against the Bharat Signature rubric
- [ ] Compare Bharat V2 vs. baseline Mistral 7B on the same 50 human-written questions
